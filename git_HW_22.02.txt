 1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

  если он в режиме untracked files(проверить с помощью команды git status)
  git checkout .               отменить измения во всех файлах
  git checkout fileName.txt    отменить измемение в указанном файле

 2.Как вернуть файл из индекса в рабочий каталог?

  git restore —stage <имя файла> - возврат файла из staged area в working directory

 3.Как отменить ошибочный commit ?

  git reset HEAD~1     - откатиться назад на 1 шаг (1 - количетсво шагов)
  git reset HEAD^      - откатиться назад на 1 шаг(^ - количетсво шагов)

 4.Как создать новую ветку и переключится в нее?

  git checkout -b <имя ветки>

 5.В чем отличение между fast-forward merge  and non fast-forfard merge? 

  fast-forward merge представляет собой ускоренное слияние. Ускоренное слияние возможно, 
  когда существует линейный путь от вершины текущей ветки до целевой ветки.
  В этом сценарии, вместо фактического слияния двух ветвей, Git интегрирует истории, т. е. предыдущие коммиты, 
  которые должны переместить вершину текущей ветки вверх к вершине целевой ветки.

  non fast-forfard merge объединяет указанную ветку с командой в текущей ветке и обеспечивает выполнение фиксации слияния, 
  даже если это fast-forward merge. Это помогает вести учет всех выполненных команд слияния в соответствующем репозитории git.

 6.Можно ли удалить ветку? Как? 

  git checkout <имя ветки для перехода> - необходимо перейти с удаляемой ветки

  git branch -d <имя ветки которую необходимо удалить>
